# Known Limitations

        Agent HQ OS is intentionally conservative. Public trust matters more than pretending the project is a complete hosted product.

        ## Current Limits

        - It does not replace engineering judgment or code review.
- It is a local operating protocol, not a hosted agent runtime.
- It depends on maintainers actually writing useful memory and telemetry.

        ## Architecture Tradeoffs

        - Fixture-first onboarding is safer than live demos, but less visually exciting.
        - Markdown-first structure works across agent tools, but does not enforce behavior unless teams run validation.
        - Approval gates slow down automation, but they make publishing, messaging, and finance workflows reviewable.

        ## Known Issues

        - Teams may over-template early workflows; start with the starter kit and add structure only when it prevents repeated mistakes.
- Agents can still ignore instructions if the boot sequence is skipped.

        ## Future Work

        - CLI generator for starter-kit installation.
- Markdown validator for boot completeness.
- Example integrations for multiple agent tools.
