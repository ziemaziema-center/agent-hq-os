# Start Here

Agent HQ OS works best when installed as a small folder structure inside an existing project.

## The 3 Concepts

1. Boot before work: read local instructions and memory.
2. Validate before claiming completion: know how success will be checked.
3. Write telemetry after work: preserve what future sessions need.

## Choose Your Tool

- ChatGPT: paste `SESSION_BOOT.md` and the relevant memory files at the start of a session.
- Claude: use `CLAUDE.md` as the tool-specific project instruction file.
- Claude Code: keep templates in the repo and ask Claude Code to follow the boot checklist before edits.
- Codex: keep `AGENTS.md` at the project root and use validation scripts.
- OpenRouter: use the same files, but keep provider credentials outside the repo.
