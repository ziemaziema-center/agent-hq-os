# v0.1.6 - Self-Improving Skill

This release adds the first review-gated self-improvement loop to Agent HQ OS.

## Added

- `skills/self_improving_skill/README.md`
- `skills/self_improving_skill/SKILL.md`
- `docs/SELF_IMPROVING_SKILL_LOOP.md`
- `templates/SKILL_IMPROVEMENT_PROPOSAL.md`

## Why It Matters

Agent HQ OS already keeps execution memory, validation, and telemetry close to the repo. The Self-Improving Skill connects those pieces into a safer improvement loop:

1. read task telemetry;
2. identify repeated failures or validated patterns;
3. draft a bounded improvement proposal;
4. require human review;
5. validate before promoting anything into memory.

This keeps the project honest. The system can improve its operating playbooks without claiming unchecked autonomous self-modification.

## Safety Boundary

The skill does not:

- read or store credentials;
- publish social posts;
- deploy infrastructure;
- activate live workflows;
- execute trades;
- rewrite future behavior without maintainer review.
